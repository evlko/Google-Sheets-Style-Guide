Google Sheets Style Guide

Idea

This is a set of guidelines that can help separate entities in your tables. It can be useful to easily find the right ones and distinguish them from each other. Also, since named functions can be exported, naming them is especially important.

The guide will be expanded and updated from time to time, so feel free to share, comment, and suggest improvements.

Rules

1 Naming Rules

1.1 Named Ranges

Uppercase with underscores: NAMED_RANGE.

This naming allows you to distinguish your ranges from all functions and keeps the appearance of the letters in the global space ('cause all of them ar uppercase).

1.2 Named Functions

Uppercase starting & with underscores: _NAMED_FUNCTION.

Named functions are worth starting with the underscore to filter them out from the standard ones. Standard functions use a dot as a separator but named functions can only have the underscore.

1.3 Function Arguments

Lowercase with underscores: arg_name.

You can use named ranges in functions, so it's better use a different case to distinguish between them and func args.
